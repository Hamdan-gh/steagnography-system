💾 SUPABASE SETUP
═══════════════════════════════════════════════════════════════════════════════

STEP 1: Create Supabase Project
───────────────────────────────────────────────────────────────────────────────
1. Go to: https://supabase.com
2. Click "New Project"
3. Choose organization
4. Enter project name
5. Generate database password
6. Choose region
7. Wait for provisioning (~2 minutes)

STEP 2: Run SQL Schema
───────────────────────────────────────────────────────────────────────────────
1. Open Supabase Dashboard → SQL Editor
2. Open file: supabase_schema.sql
3. Copy all contents
4. Paste in SQL Editor
5. Click "Run"
6. Wait for success message

This creates:
  • users table
  • stego_images table
  • extraction_logs table
  • audit_logs table
  • All indexes
  • All RLS policies
  • All triggers

STEP 3: Create Storage Buckets
───────────────────────────────────────────────────────────────────────────────
1. Go to Storage in Supabase Dashboard
2. Click "New Bucket"
3. Create these 3 buckets:

   Bucket 1:
     Name: images
     Public: YES ✓

   Bucket 2:
     Name: audio
     Public: NO

   Bucket 3:
     Name: stego-images
     Public: YES ✓

STEP 4: Get Credentials
───────────────────────────────────────────────────────────────────────────────
1. Go to Project Settings → API
2. Copy:
   • Project URL (https://xxx.supabase.co)
   • anon/public key (long string starting with "eyJ...")

STEP 5: Update .env File
───────────────────────────────────────────────────────────────────────────────
Edit .env file in project root:

VITE_SUPABASE_URL=https://your-project.supabase.co
VITE_SUPABASE_ANON_KEY=eyJhbGc...your-key-here
VITE_PROCESSING_ENGINE_URL=http://localhost:5000

STEP 6: Restart Frontend
───────────────────────────────────────────────────────────────────────────────
• Press Ctrl+C in terminal
• Run: npm run dev
• Open: http://localhost:5173

🐛 TROUBLESHOOTING
═══════════════════════════════════════════════════════════════════════════════

ISSUE: "Row Level Security" Error
───────────────────────────────────────────────────────────────────────────────
SOLUTION: Run the full SQL schema (supabase_schema.sql)

ISSUE: Can't Upload to Storage
───────────────────────────────────────────────────────────────────────────────
SOLUTION: 
  1. Check bucket exists
  2. Check bucket is public (for images/stego-images)
  3. Check user is authenticated

ISSUE: Sidebar Doesn't Show on Mobile
───────────────────────────────────────────────────────────────────────────────
SOLUTION: Look for hamburger menu (☰) in top-left corner

ISSUE: Page Not Responsive
───────────────────────────────────────────────────────────────────────────────
SOLUTION: 
  1. Clear browser cache (Ctrl+Shift+Delete)
  2. Restart dev server (Ctrl+C, npm run dev)
  3. Hard refresh page (Ctrl+F5)

ISSUE: Timeout Still Occurring
───────────────────────────────────────────────────────────────────────────────
SOLUTION: See TIMEOUT_SOLUTION.md
  • Use 10 generations, 10 population
  • Use smaller files (< 1MB image, < 500KB audio)
